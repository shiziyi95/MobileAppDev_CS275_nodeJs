                    2. Chapter 12

                      Step-by-step solution
                      1. Step 1 of 5

                        Consider the given TINY server. Now consider the TINY serve_static, given in figure 11.33, which is used to serve static content to a client.

                        • In serve_static, it uses mmap and rio_writen to copies the requested file to the connected descriptor.

                        • Now user can use malloc, rio_readn and rio_writen to do same (to copies the requested file to the connected descriptor), instead of mmap and rio_writen.

                        Comment
                      2. Step 2 of 5

                        Consider the following modified content of serve_static function:

                        void serve_static(int fd, char *filename, int filesize)

                        {

                        int srcfd;

                        char *srcp, filetype[MAXLINE], buf[MAXBUF];

                        /* Send response headers to client */

                        get_filetype(filename, filetype);

                        sprintf(buf, "HTTP/1.0 200 OK\r\n");

                        sprintf(buf, "%sServer: Tiny Web Server\r\n", buf);

                        sprintf(buf, "%sContent-length: %d\r\n", buf,

                        filesize);

                        sprintf(buf, "%sContent-type: %s\r\n\r\n", buf,

                        filetype);

                        //The rio writen function transfers strlen(buf) bytes

                        //from location buf to descriptor fd.

                        Rio_writen(fd, buf, strlen(buf));

                        // Open filename for reading and gets its descriptor

                        srcfd = Open(filename, O_RDONLY, 0);

                        The following statements shows the modified part.

                        //Starts at address srcp

                        srcp =malloc(srcfd);

                        //The rio_readn function transfers strlen(buf) bytes

                        //from the current files position of descriptor fd to

                        //memory location buf

                        rio_readn(fd, srcp, filesize);

                        //Once user takes a files to memory, user no longer

                        //need its descriptor, so user close it

                        Close(srcfd);

                        //The actual transfer of file to the client is done

                        //here. Here the rio_writen function copies the file

                        //size bytes starting at location srcp to

                        //the client's connected descriptor.

                        Rio_writen(fd, srcp, filesize);

                        //free the virtual memory area

                        Munmap(srcp, filesize);

                        }

                        Comment
                      3. Step 3 of 5

                        The function definitions of malloc, rio_readn and rio_writen are shown below:

                        //Defined malloc function here.

                        void *Malloc(size_t size)

                        {

                        void *p;

                        if ((p = malloc(size)) == NULL)

                        unix_error("Malloc error");

                        return p;

                        }

                        //Define rio_readn function in which it return ssize_t means

                        // it may return a negative value

                        ssize_t rio_readn(int fd, void *usrbuf, size_t n)

                        {

                        size_t nleft = n;

                        ssize_t nread;

                        char *bufp = usrbuf;

                        //Check for the positive value

                        while (nleft > 0)

                        {

                        if ((nread = read(fd, bufp, nleft)) < 0)

                        {

                        /* interrupted by sig handler return */

                        if (errno == EINTR)

                        /* and call read() again */

                        nread = 0;

                        else

                        /* errno set by read() */

                        return -1;

                        }

                        else if (nread == 0)

                        /* EOF */

                        break;

                        nleft -= nread;

                        bufp += nread;

                        }

                        /* return >= 0 */

                        Comment
                      4. Step 4 of 5
                        return (n - nleft);

                        }

                        //Define rio_writen function in which it return ssize_t means

                        // it may return a negative value

                        ssize_t rio_writen(int fd, void *usrbuf, size_t n)

                        {

                        size_t nleft = n;

                        ssize_t nwritten;

                        char *bufp = usrbuf;

                        while (nleft > 0)

                        {

                        if ((nwritten = write(fd, bufp, nleft)) <= 0)

                        {

                        /* interrupted by sig handler return */

                        if (errno == EINTR)

                        /* and call write() again */

                        nwritten = 0;

                        else

                        /* errno set by write() */

                        return -1;

                        }

                        nleft -= nwritten;

                        bufp += nwritten;

                        }

                        return n;

                        }

                        Comment
                      5. Step 5 of 5

                        As mmap is used previously, to create a new mapping in the virtual address space of the calling process.

                        • The addr is used to specify the starting address of the new mapping. The length of the mapping is specified using length variable.

                        • As mmap doesn’t read the file actually, it is only used to maps it to address space. In the same way malloc is used to mapping of the address space to the memory.

                        • The rio_readn function transfers strlen(buf) bytes from the current files position of descriptor fd to memory location buf.

                        • The rio_writen function copies the file size bytes starting at location srcp to the client's connected descriptor.
